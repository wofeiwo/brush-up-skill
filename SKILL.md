# ✨ Brush-Up Skill

> 持续自我改进的 OpenClaw Agent Skill

**版本**: v2.1 (2026-03-12)

---

## 核心约束 (Prime Directives)

1. **未经授权，不执行任何危险操作**
2. **未经授权，不泄露个人隐私**
3. **人类是最终决策者**
4. **Token 效率优先**

---

## 核心原则

**brush-up 是一个流程指导技能，不是自动化工具！**

- ✅ Skill 提供流程框架、工具列表、模板
- ❌ Skill 不代替 Agent 执行
- ✅ Agent 必须全程参与、自主决策、主动思考

---

## 输入输出

### 输入
- `--topic`: 议题名称（必需）
- `--agent`: 从属 Agent（可选，自动检测）
- `--categories`: 议题类别（可选，自动推荐）
- `--frequency`: 循环频率 daily/weekly/monthly（可选，自动推荐）
- `--mode`: 流程模式 light/standard/full（可选，自动推荐）
- `--channel`: 投递渠道（多 channel 环境必需）

### 输出
- P1 信息收集报告
- P2 价值筛选结果
- P3 反思洞察
- P5 分级汇报（含提案和风险分级）

---

## 基本流程 (P1-P7)

```
P1 信息收集 → P2 价值筛选 → P3 反思洞察 → P4 推演验证 → P5 分级汇报 → P6 实施沉淀 → P7 记录比对
```

| 阶段 | 时间 | 核心任务 |
|------|------|----------|
| P1 | 08:00 | 多源扫描，收集信息 |
| P2 | 08:15 | 黑名单否决，质量评分 |
| P3 | 08:30 | 深度反思，提炼洞察 |
| P4 | 09:00 | 思想实验，小范围测试 |
| P5 | 21:00 | 生成报告，请求授权 |
| P6 | 批准后 | 执行变更，记录经验 |
| P7 | 完成后 | 更新指标，趋势分析 |

---

## 强制反思提示 (必须回答)

**P1 - 信息收集策略**:
- 选择的信息源？
- 为什么选择这些源？
- 预期收获什么？

**P3 - 深度反思三问**:
- 哪条信息最意外？为什么？
- 这个洞察能解决什么实际问题？
- 一句话总结核心发现？

**P5 - 提案价值自检**:
- 3 点优势？
- 2 点风险？
- 用户视角关心什么？

|------|------|------|

|--------|------|------|------|

| 重构/架构/设计 | dev-tools | weekly | **full** |
| 优化/改进/机制 | openclaw-ecosystem | weekly | standard |
| 交易/量化/市场 | finance | daily | standard |
| 学习/研究/AI | ai-tech | weekly | standard |

## 关键提醒

- ⚠️ 多 channel 环境必须使用 `--channel` 参数
- ⚠️ Agent 只是被动执行则失去意义
- ⚠️ 必须深度思考每个阶段，不能跳过反思提示
